# 코드 리뷰 - 커밋되지 않은 변경사항

커밋되지 않은 변경사항에 대해 종합적인 코드 리뷰를 수행합니다.

## 수행 절차

### 1단계: 변경사항 수집

다음 명령을 실행하여 변경사항을 파악합니다:

```bash
git status
git diff --stat
git diff
git diff --staged
```

### 2단계: 변경 파일 분석

각 변경된 파일에 대해 다음을 분석합니다:

1. **파일 컨텍스트 파악**: 변경된 파일의 전체 구조와 역할 이해
2. **변경 범위 확인**: 추가/수정/삭제된 라인 파악

### 3단계: 코드 리뷰 체크리스트

다음 항목들을 기준으로 리뷰합니다:

#### 버그 및 로직 오류
- 널 포인터 예외 가능성
- 경계 조건 처리 누락
- 비동기 처리 이슈
- 리소스 누수 (열린 연결, 스트림 등)
- 잘못된 조건문/반복문 로직

#### 코드 품질
- 함수/메서드 길이 및 복잡도
- 중복 코드
- 네이밍 컨벤션 준수
- 매직 넘버/문자열 사용
- 불필요한 코드 (데드 코드, 주석 처리된 코드)

#### 보안
- 입력값 검증 누락
- SQL 인젝션 가능성
- 민감 정보 노출 (비밀번호, API 키 등)
- 인증/인가 처리 미흡

#### 성능
- N+1 쿼리 문제
- 불필요한 반복/연산
- 캐싱 기회
- 대용량 데이터 처리 시 메모리 이슈

#### Kotlin/Java 특화 검토 (해당 시)
- Nullable 처리 적절성
- 코루틴/비동기 처리 패턴
- 예외 처리 전략
- 의존성 주입 적절성

### 긍정적인 변경사항

잘 작성된 부분이나 좋은 패턴이 있다면 언급합니다.

---

### 5단계: 수정 진행 여부 확인

**중요**: 리뷰 결과 출력 후, 반드시 AskUserQuestion 도구를 사용하여 사용자에게 어떤 이슈를 수정할지 물어봅니다.

AskUserQuestion 호출 예시:
- question: "어떤 이슈를 수정하시겠습니까?"
- header: "수정 선택"
- multiSelect: true
- options: 발견된 각 이슈를 옵션으로 제공 (예: "1. Nullable 처리 누락", "2. N+1 쿼리 문제" 등)

사용자가 선택한 이슈에 대해서만 코드 수정을 진행합니다. 선택되지 않은 이슈는 수정하지 않습니다.

---

## 주의사항

- 린터나 컴파일러가 잡을 수 있는 사소한 이슈는 제외합니다
- 기존에 존재하던 문제는 이번 리뷰 범위에서 제외합니다
- 의도적인 변경으로 보이는 부분은 확인 후 언급합니다
- 스타일 관련 지적은 프로젝트 컨벤션에 따릅니다
- 사용자 선택 없이 임의로 코드를 수정하지 않습니다
